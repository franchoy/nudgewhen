# Code of Conduct

This Code of Conduct applies to the NudgeWhen project repository, including all issues, pull requests, discussions, and any other project-managed communication spaces.

## Our expectations

Every participant in this project is expected to behave respectfully, constructively, and inclusively. This includes:

- Using welcoming and inclusive language.
- Being respectful of differing viewpoints and experiences.
- Gracefully accepting constructive criticism.
- Focusing on what is best for the community and the project.
- Showing empathy towards other community members.

## Unacceptable behaviour

The following behaviour is unacceptable:

- Harassment in any form, including unwelcome attention, intimidation, stalking, or following.
- Discriminatory language or actions related to gender, gender identity, sexual orientation, disability, physical appearance, body size, race, age, nationality, religion, or any other personal characteristic.
- Threats, violent language, or incitement to violence.
- Doxxing or publishing another person's private information without their explicit permission.
- Deliberate disruption of issues, pull requests, discussions, or other project spaces.
- Spam, off-topic commentary, or repetitive low-effort content.
- Any conduct that a reasonable person would consider inappropriate in a professional or collaborative setting.

## Scope

This Code of Conduct applies within all project-managed spaces, including:

- Repository issues and pull requests.
- Project-managed communication channels, if any are added in the future.
- Any other space where an individual is representing the project.

## Moderation

The project is currently maintained by a single maintainer. The maintainer may moderate project spaces to keep them welcoming and on-topic. Proportionate actions may include:

- Editing or removing comments, issues, or pull requests.
- Hiding or locking threads.
- Closing or rejecting issues or pull requests.
- Restricting participation in project spaces.
- Any other action the maintainer considers appropriate to the situation.

## Reporting

If you experience or witness behaviour that violates this Code of Conduct, the maintainer is the contact point. Two routes are available:

- **GitHub platform tools.** Use GitHub's built-in reporting and blocking mechanisms on the relevant issue, pull request, comment, or user profile where applicable.
- **Private contact fallback.** Open a minimal public issue that requests a private contact route. The issue body must contain no names, allegations, evidence, screenshots, personal details, or other sensitive information. It should request a private channel only.

There is no guarantee of confidentiality, outcome, response time, or appeal process. The project is single-maintainer and experimental.

## Security vulnerabilities are not covered here

Security vulnerabilities are reported through `SECURITY.md`, not through the conduct-reporting routes described above. Do not file security vulnerabilities through the conduct-reporting route.

## No moderation team

The project has a single maintainer, not a moderation team, committee, or multi-maintainer body. There is no escalation path beyond the maintainer.
