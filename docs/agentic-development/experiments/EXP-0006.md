- it treated correctly negated compatibility statements as broad compatibility claims;
- one version checked the wrong Stage 1 corrective-prompt value;

Correction does not erase the prior closure deviations.

### Offline-command stop-condition deviation

- The successful retry does not erase the unauthorized continuation.

The initial exit `1` was not an Android compilation failure; it was a missing-directory precondition failure in the shell pipeline.
