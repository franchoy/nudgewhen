### Experiment finding (failed pre-Build snapshot capture)

The failed pre-Build snapshot capture is itself an experiment finding. The planned protocol expected a numeric pre-Build snapshot to anchor the Build-stage delta. When the maintainer is unable or unavailable to capture a snapshot at the planned moment, the protocol should fall back to recording the snapshot as `Not available` and explicitly mark the Build-stage delta as `Not computable`, which is what was done here. This is recorded as a candidate process improvement.

**Candidate process improvement:** The experiment protocol should be extended to define an explicit, neutral fallback for a missing pre-Build snapshot (for example, when the maintainer has not yet reached the OpenCode Go console). The current protocol already permits `Not available` placeholders, but a step-by-step fallback for the snapshot moment itself should be documented in a future Phase 0 amendment or in the Phase 6 evaluation summary.
