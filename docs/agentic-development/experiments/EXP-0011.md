## Direct-versus-maintainer-versus-inference labelling

- **Direct evidence** is recorded when the source is the experiment record itself.
- **Maintainer classification (later)** is recorded when the source is the maintainer's `Accepted` / `Pending maintainer review` / `Pending maintainer input` line, a maintainer-authored classification section in the record, or a classification supplied in the Phase 6 authorization. Where no later maintainer classification exists, the field is the literal `Not recorded` token and is not invented.
- **Phase 6 inference** is recorded when the source is the present evaluation draft and is not a direct quote or close paraphrase of a record. The Phase 6 inference column does not carry a record-evidence label; direct facts live in the technical-result, execution-scope, and direct-deviations columns.

## Control-gap method

- For each repeated lesson, the current state is classified into one of four categories: existing prose guidance; repository-observable deterministic check; execution-transcript-only behavior; runtime/tool-required behavior.
- A repository validator may deterministically inspect only such things as exact changed path sets; hashes and byte ranges; tracked and untracked state; temporary or bytecode files visible in the workspace; links and document structure; status synchronization; committed configuration and workflow structure.
- A repository validator does not inspect: whether an agent continued after FAIL; whether it used a proxy test during a conversation; whether it weakened an assertion; whether it overstated an interactive report; whether it retried a remote mutation; whether it printed excessive transient output.
